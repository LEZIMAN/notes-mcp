## 1. 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| **MCP server** | FastMCP(Python) | 三大原语 + lifespan 建库 + 双传输(stdio/HTTP) |
| **检索** | Chroma + bm25s + jieba | 语义向量 + 关键词(BM25)+ RRF 融合 |
| **LLM / Embedding** | ollama | `qwen3:8b`(生成)+ `bge-m3`(1024 维嵌入) |
| **Web 后端** | FastAPI + uvicorn | 作 MCP client,通过 HTTP 连 notes-mcp,给前端 REST API |
| **Web 前端** | React + Vite + TypeScript | 图形对话界面(聊天 + 检索/讲解/复习展示) |
| **测试** | pytest + pytest-asyncio + FastMCP in-memory | 单元 + 集成,e2e 标 integration |

## 2. 服务架构与端口

```
React 前端 (5173)  ──HTTP──►  FastAPI 后端 (8000)  ──MCP/HTTP──►  notes-mcp server (8765)
                                                                      │
                                                      Chroma + BM25 + SQLite
                                                                      │
                                                                  ollama (11434)
```

| 服务 | 默认端口 | 作用 | 启动 |
|---|---|---|---|
| ollama | 11434 | LLM 生成 + embedding | `ollama serve`(系统服务) |
| **notes-mcp server** | 8765 | MCP 协议端点(`/mcp/`),供 client 调 | `python -m notes_mcp serve --transport http` |
| **FastAPI 后端** | 8000 | Web API(作 MCP client,连 8765) | `uvicorn web.backend.main:app --port 8000` |
| **React 前端** | 5173 | 图形对话界面(Vite dev) | `npm run dev` |

> 三层 client 链:前端 → FastAPI(客户端 A)→ notes-mcp server → 数据。notes-mcp 仍是唯一的知识源,任意 client(Claude Desktop / FastAPI / 自写 agent)都能接。

## 4. 环境变量(`.env.example`)

```bash
# ===== 笔记数据 =====
# 笔记目录(支持多个,分号 ; 分隔;笔记分散在多处也行)
NOTES_DIR=d:/Learn/AI/笔记

# ===== ollama(生成 + 嵌入)=====
OLLAMA_BASE_URL=http://127.0.0.1:11434/v1   # 用 127.0.0.1 避 IPv6
OLLAMA_MODEL=qwen3:8b
OLLAMA_API_KEY=ollama
EMBED_MODEL=bge-m3

# ===== 检索 / 切块 =====
TOP_K=5
CHUNK_SIZE=300
OVERLAP=50

# ===== 存储 =====
CHROMA_PATH=./data/chroma
SQLITE_PATH=./data/index_state.db

# ===== notes-mcp server(HTTP transport,供 Web 后端 / 外部 client 连)=====
MCP_HTTP_HOST=0.0.0.0
MCP_HTTP_PORT=8765

# ===== Web 后端(FastAPI)=====
WEB_BACKEND_HOST=0.0.0.0
WEB_BACKEND_PORT=8000
FRONTEND_ORIGIN=http://localhost:5173        # CORS 白名单(Vite dev)

# ===== 日志 =====
LOG_LEVEL=INFO
```

## 6. 运行(完整步骤)

```bash
# 0. 前提:ollama 已装 + 拉模型
ollama serve
ollama pull qwen3:8b
ollama pull bge-m3

# 1. Python 环境
cd d:/Learn/AI/项目/notes-mcp
python -m venv venv
venv/Scripts/python -m pip install -r requirements-dev.txt

# 2. 配置
cp .env.example .env
#   编辑 NOTES_DIR 指向你的笔记目录

# 3. 首次建库(扫笔记 → 切块 → embed → Chroma+BM25)
venv/Scripts/python -m notes_mcp index

# —— 以下三个服务各开一个终端 ——

# 4a. notes-mcp server(HTTP,供 Web 后端连)
venv/Scripts/python -m notes_mcp serve --transport http

# 4b. Web 后端(FastAPI,作 MCP client)
venv/Scripts/uvicorn web.backend.main:app --port 8000

# 4c. Web 前端(Vite dev)
cd web/frontend
npm install
npm run dev            # → 浏览器开 http://localhost:5173
```

> 也可纯命令行用:`python -m notes_mcp serve --transport stdio` + MCP inspector / Claude Desktop,不启 Web。

## 7. 开发节奏(建议分阶段)

1. **核心 server**:`config/embedder/indexer/search/server` + 测试(先把 MCP 知识库跑通,inspector 验证)
2. **消费侧 agent**(命令行):验证多 server 动态发现
3. **HTTP transport**:notes-mcp 跑 HTTP,确认 `/health` + client 连通
4. **Web 后端**:FastAPI 作 MCP client,REST 给前端
5. **Web 前端**:React 对话界面,调 FastAPI
6. 加分:CLI / 更多 Prompt / 测试覆盖 / 部署

> 各模块具体设计在实现前再讨论(用户偏好)。
