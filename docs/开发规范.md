## 2. 模块边界与依赖方向(单向、不循环)

```
config ◄ embedder ◄ indexer ◄ search ◄ server ◄ cli
```

- **业务层**(`indexer` / `search`)**不 import fastmcp** —— 纯 Python,可脱离 MCP 直接单测。
- **协议层**(`server`)import 业务层,用 `@mcp.tool` 等装饰器**薄封装**(只转发,不含业务逻辑)。
- `cli` 组装并启动 server。

## 3. 可测性设计(核心)

让代码「能被自动测试」的三条铁律:

### 3.1 业务逻辑下沉为类 / 纯函数
```python
# search.py —— 纯逻辑,不碰 MCP
def rrf_fuse(ranked_lists: list[list[str]], k: int = 60) -> list[tuple[str, float]]:
    """RRF 融合多路排序结果。纯函数,直接单测。"""
    ...

class Searcher:
    def __init__(self, collection, bm25, embedder):  # 依赖注入
        ...
    def search(self, query: str, top_k: int = 5) -> list[Hit]:
        ...
```

### 3.2 依赖注入(便于 mock)
- `Indexer` / `Searcher` 通过构造参数接收 `embedder` / `collection` / `bm25` —— 测试时传 **fake**,不依赖真实 ollama。
- ❌ 不在函数内 `embedder = OllamaEmbedder()`(无法替换)。

### 3.3 server 是薄封装
```python
# server.py —— 只转发,不含逻辑
@mcp.tool()
def search_notes(query: str, top_k: int = 5) -> str:
    """检索学习笔记(hybrid)。"""           # ← docstring 即 MCP description
    hits = ctx.lifespan_context["searcher"].search(query, top_k)  # ← 转发
    return format_hits(hits)
```

> 由此测试分两层:**业务层直接单测(快、纯)**;**协议层用 FastMCP in-memory client 测(测三大原语注册 + 调用)**。

## 4. 代码风格

| 项 | 要求 |
|---|---|
| 类型注解 | 函数签名**必填**(`query: str`, `-> str`) |
| docstring | Google 风格、中文;MCP tool/resource/prompt 的 docstring 会当 description,必须写清 |
| 注释 | **教学型详细中文注释**(给初学者看懂) |
| 配置 | 外置 `.env` + `config.py`,**绝不硬编码** |
| 路径 | 用 `pathlib.Path`,不拼字符串 |
| 日志 | `logging` 模块;**server 内禁 `print`**(污染 stdio 协议) |
| 异常 | 自定义异常分层(`IndexerError` / `SearchError`),友好消息 |
| 风格 | `ruff` / `black`(可选,统一格式) |

## 5. 测试策略(金字塔)

```
   ┌───────────┐
   │  e2e(少) │  真实 ollama + stdio   @pytest.mark.integration
   ├───────────┤
   │ 集成(中) │  FastMCP in-memory Client 调 tool/resource/prompt(用 fake embedder)
   ├───────────┤
   │ 单元(多) │  Indexer / Searcher / rrf_fuse / chunk(纯逻辑,毫秒级)
   └───────────┘
```

### 5.1 工具链
- `pytest` + `pytest-asyncio`(`asyncio_mode = "auto"`,免装饰)
- **FastMCP in-memory 测试**(官方推荐):`async with Client(server) as c: await c.call_tool(...)` —— 同进程跑真实 MCP 协议,毫秒级、可断点、不部署。
- `inline-snapshot`(复杂结构断言,可选)、`dirty-equals`(动态值,可选)

### 5.2 `pyproject.toml` 配置
```toml
[tool.pytest.ini_options]
asyncio_mode = "auto"
markers = [
  "integration: 需要真实 ollama/网络,慢(CI 可跳过)",
  "needs_ollama: 需要 ollama 服务在跑",
]
```

### 5.3 `conftest.py` 公共 fixtures
| fixture | 作用 |
|---|---|
| `tmp_notes` | 临时笔记目录(几篇 .md,测完自动清) |
| `fake_embedder` | 返回固定向量的假 embedder(**不依赖 ollama**) |
| `memory_chroma` | 内存 Chroma client(EphemeralClient) |
| `indexer` / `searcher` | 用上面三个组装的业务对象 |
| `server` | 装配好的 FastMCP server(注入上面的 searcher) |
| `client` | `async with Client(server)` 的 in-memory client |

> ⚠️ 官方提醒:**别在 fixture 里 open client**(event loop 问题),在 test 内 `async with Client(server) as c`。

### 5.4 测试原则(摘自 FastMCP 官方)
- **单行为单测**:一个测只验一件事,失败时一眼知道哪坏了。
- **自包含**:每个测自己 setup,可任意顺序/并行/隔离跑。
- **命名表意**:`test_search_notes_returns_provenance`(看名知测什么)。
- **断言具体**:带失败消息 `assert x == y, f"..., got {y}"`。

## 6. 测试示例

**单元(RRF 纯函数)**:
```python
def test_rrf_fuse_prefers_docs_in_both_lists():
    sem = ["a", "b", "c"]            # 语义路排名
    bm  = ["b", "d", "a"]            # 关键词路排名
    fused = rrf_fuse([sem, bm], k=60)
    # b 在两路都靠前 → 应排第一
    assert fused[0][0] == "b"
```

**集成(server via in-memory client,用 fake embedder)**:
```python
async def test_search_notes_tool_returns_hits_with_source(server):
    async with Client(server) as c:
        res = await c.call_tool("search_notes", {"query": "RAG", "top_k": 3})
    assert res.data                     # 有结果
    assert "来源" in res.data           # 带溯源
```

## 7. 运行

```bash
pytest                              # 全跑(单元 + 集成,用 fake,快)
pytest -m "not integration"         # 跳过需真实 ollama 的
pytest tests/test_search.py         # 单文件
pytest --cov=notes_mcp              # 覆盖率
```
